§ 04 — Guía de Imágenes

Imágenes GitHub-Ready

Cómo subir fotos que no rompan el JSON de GitHub.

⚠ Por qué NO subir fotos directamente

Cuando subes una foto desde el computador sin un servicio externo, se guarda como base64 — texto larguísimo dentro del JSON. Una sola foto puede hacer que el archivo pese +2MB, GitHub lo rechaza o la tienda carga muy lento.

① Opción recomendada: Imgur (automático)

1

En "Agregar Producto", selecciona la pestaña ⬆ Subir a Imgur

2

Haz click en el recuadro y selecciona tu foto. El gestor la sube automáticamente a Imgur.

3

El link se guarda solo. El JSON solo almacena la URL, no la foto.

② Opción manual: Imgur.com

1

Ve a imgur.com → click en "New post" → sube tu foto

2

Click derecho sobre la imagen → "Copiar dirección de imagen"

3

El link debe verse así: https://i.imgur.com/AbCdEf1.png

4

Pega ese link en la pestaña 🔗 Pegar Link del formulario

③ Limpiar base64 existente

Si ya tienes productos con base64, el gestor los detecta con ⚠. Edita cada uno, quita la imagen actual y reemplázala con un link de Imgur.

Esto eliminará las imágenes base64 de todos los productos. Deberás reemplazarlas con links.

§ 05 — GitHub Sync

Sincronización con GitHub

Carga el catálogo directamente desde un repositorio público de GitHub.

Acciones del catálogo

Cargar desde GitHub Raw URL

Pega el link raw.githubusercontent.com de tu JSON en el repositorio.

1

Sube el JSON exportado a tu repositorio de GitHub

2

Abre el archivo en GitHub → click en "Raw"

3

Copia la URL — empieza con raw.githubusercontent.com

4

Pega la URL arriba y haz click en Cargar

ⓘ Flujo recomendado con GitHub

1

Exporta el JSON desde este gestor — sin base64

2

Súbelo a tu repo en GitHub como spc-catalogo.json

3

Tu tienda / web lee el JSON del raw URL directamente

4

Para actualizar: exporta de nuevo y sube el nuevo archivo
